\title{alignment}

\subject{Issues:}

\startitemize
\item For default alignment issues, see \type{testlinenotes-001}.  {\bf high}
\item \type{distance=} seems to have no effect upon note numerals.  {\bf high}
\item \type{width=} seems to have no effect upon note numerals.  {\bf high}
\item \type{[align=r2l]} affects note numerals but not line numerals. Shouldn't this be symmetric?  {\bf high}
\item \type{align=r2l} affects the first line of the next buffer; see p.~12.  {\bf low}
\item \type{separator={ : }} has no apparent effect. What is its function in linenotes?  {\bf medium}
\item Can we have eg \type{[left={[},right={]}]} for note numerals as well?  {\bf medium}
\item \type{next=\autoinsertnextspace}. The function of this option is not clear. {\bf high}
% \item
\stopitemize
